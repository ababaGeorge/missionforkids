## Key Interactions to Verify

- Parent creates task → taskInstance auto-created for each child profile
- Child taps task → camera opens → photo taken → uploaded to Firebase Storage → taskSubmission created
- Parent approves submission → points atomically awarded (wallet + transaction + instance status)
- Child redeems reward → points atomically deducted (wallet + transaction + order created)
- Parent rejects submission → child sees "Try again" badge → can resubmit once
- Second rejection → task instance marked missed (no punishment)
- Parent cancels reward order → points refunded atomically
- Parent generates invite code → code stored with 24hr expiry → child joins family
- Profile switcher → navigation changes between child/parent tab layouts

## Edge Cases

- Photo upload fails 3 times → queued locally → retried when online
- Concurrent double-approval of same submission (idempotency)
- Concurrent double-redemption of same reward (race condition → only one succeeds)
- Redeem with exactly enough points (boundary: balance == cost)
- Expired invite code (>24hr) → rejected
- Already-used invite code → rejected
- Already-cancelled reward order → cancel is no-op
- Already-fulfilled reward order → cancel blocked
- Empty states: no tasks, no submissions to review, no rewards, no transactions
- Zero-point task (validation: should be blocked?)
- Child with no wallet yet (auto-create on first task approval)
- i18n: all screens render correctly in zh-TW and en

## Critical Paths

- Core loop E2E: Parent creates task → child submits photo → parent approves → child earns points → child redeems reward → parent fulfills → completed
- Rejection E2E: Child submits → parent rejects → child resubmits → parent approves → points awarded
- Onboarding E2E: Parent signs up → creates family → creates task → generates invite code → child joins → child sees tasks
- Offline resilience: Child submits photo while offline → photo queued → app goes online → photo uploaded → submission created
- Profile switching: Parent approves on shared device → switches to child profile → child sees updated points
