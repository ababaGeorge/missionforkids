> 版本：v1.1｜最後更新：2026-03-19

## 一、產品定位

以遊戲化任務與點數獎勵，幫助孩子建立原子習慣的家庭 App。  
核心理念：**正向激勵，不做懲罰機制。**  
初期目標：家庭版 MVP。社群版（班級/朋友圈）為第二階段，時程待定。

---

## 二、使用者與帳號結構

| 決策項目 | 結論 |
|---|---|
| 角色種類 | 家長 / 孩子（初版），老師（第二階段） |
| 家庭人數 | 不限，參考 Apple Family 模式 |
| 孩子帳號 | 獨立帳號，支援 Apple ID / Google ID 登入 |
| 家長帳號 | 獨立帳號，不帶點數，可被多個家庭邀請 |
| 最小年齡 | 6 歲（小學一年級），UI 需支援基本語義理解與操作 |
| 帳號轉移 | 孩子帳號可跨家庭轉移（點數與歷史紀錄隨行）；家長不需轉移，重新被邀請即可 |
| 離婚/分家 | 孩子帳號轉移到新家庭；家長另行加入 |

**資料結構核心：**
```
Family（家庭）
  └─ Members［家長 / 孩子，含角色］

Group（群組）← 第二階段
  └─ Members
  └─ Tasks

User（用戶）
  └─ 可屬於多個 Family 或 Group（角色不同）
```

## 三、任務系統

### 任務類型

設計原則：以家長操作流程為主軸，不用分類名稱，用「頻率」和「截止日」來定義任務。

**一次性任務**
- 必填：任務名稱、點數、截止日期
- 標籤（語意用，不影響邏輯）：**挑戰**（如比賽、考試）/ **臨時**（如臨時交辦事項）
- 節慶任務 = 一次性任務 + 截止日期，不需獨立類型

**週期任務**
- 週期選項：每日 / 每週 / 每月 / 每學期
- 每日、每週、每月：選定第一個起始日，後續自動計算
- 每學期：預設結算日為 **1/31**（上學期）和 **6/30**（下學期），可手動調整

### 完成歷程與統計

- 每個週期的完成／未完成狀態都會記錄，不補做也不懲罰，直接進入下一期
- 家長與孩子均可查看歷程：任務完成率、連續完成天數、點數趨勢等
- 此為數據基礎，日後可延伸為成長報告

### 任務狀態機

```
建立 → 發佈 → 進行中 → 孩子提交（含照片）
                              ↓
                        AI 審核 / 半自動 / 人工
                              ↓
                        通過 → 點數發放
                              ↓
                        未通過 → 重拍提示 / 轉人工審核
```

### 審核機制（每個任務可獨立設定）

| 模式 | 說明 |
|---|---|
| AI 審核 | 孩子拍照，AI 判斷「有沒有做」（方向一：完成度辨識，非品質評分） |
| 半自動 | AI 通過即核准；家長有時間窗口（建議 24 小時）可推翻 |
| 人工審核 | 家長手動審核 |

**AI 審核邊界（初版）：**
- ✅ 判斷「有沒有做」：書桌乾淨了嗎、作業本有寫字嗎
- ❌ 不判斷「做得好不好」：數學對不對、字夠不夠工整（預留架構，第二階段）
- 若 AI 多次判斷未完成，提示孩子「畫面模糊，請重拍」，或自動轉人工審核

### 其他任務規則

- 孩子可以自己**提議任務**，家長審核後決定是否開放及點數設定
- 團隊任務：每位完成的成員**各自獲得全額點數**（不均分，帶人也有成就感）
- 初版**無懲罰/扣點機制**

## 四、點數系統

| 決策項目 | 結論 |
|---|---|
| 點數過期 | 不過期 |
| 孩子互轉 | 不開放 |
| 對應真實金錢 | 純虛擬；提供參考換算建議給家長（例：100點 ≈ NT$100） |
| 家長直接給點 | 可以，但需填寫名目（自由輸入 + 預設常用名目） |
| 扣點機制 | 初版不做 |

**預設名目範例：** 生日獎勵、特別表現、旅行紀念、節日紅包、鼓勵獎勵

## 五、願望清單 / 商城

| 決策項目 | 結論 |
|---|---|
| 商品建立者 | 家長設定為主；提供預設範本幫助家長上手 |
| 孩子提案 | 初版不做（可實體提案），第二版加入 |
| 兌換流程 | 孩子送出訂單 → 家長審核 → 交付 → 孩子點擊「確認收到」（或逾時自動完成） |
| 商品類型 | 實體（玩具、零食）＋ 虛擬（遊戲時間、選一部電影） |
| 未來整合 | 預留與 Apple Screen Time API 整合的架構（多 30 分鐘遊戲時間自動執行） |

## 六、技術方向

| 層面 | 決策 |
|---|---|
| 平台 | iOS（主）＋ Android，上架 App Store ＆ Google Play |
| 框架 | React Native 或 Flutter（待定，開發資源確認後決定） |
| 後端 | Firebase 或 Supabase（MVP 階段，快速迭代優先） |
| 認證 | Sign in with Apple（必要）＋ Google Sign-In |
| 離線能力 | 需支援基本離線操作（任務查看、打卡），同步衝突策略待定 |
| 兒童資料法規 | 需符合 COPPA（美）/ GDPR-K（歐），13歲以下帳號由家長建立並同意 |
