版本：v1.0　｜　最後更新：2026-03-20　｜　所屬專案：任務獎勵 App

> 本文件為任務系統的詳細邏輯決策，供開發參考使用。總覽請見《任務獎勵 App 專案決策文件》。

## 一、任務狀態機與流程

### 1-1　設計原則

- 建立即發佈，不設草稿狀態。建立錯誤時刪除重建。
- 週期任務每一期為獨立任務實例，到期自動建立下一期。

### 1-2　一次性任務狀態

| 狀態 | 說明 | 下一狀態 |
|---|---|---|
| 進行中 | 任務建立後即進入此狀態 | 已提交 / 已逾期 |
| 已提交（審核中） | 孩子提交照片後 | 已完成 / 重拍提示 |
| 已完成 | 審核通過，點數發放 | — |
| 已逾期 | 截止日＋補交期後未提交 | — |

### 1-3　週期任務狀態機

每一期視為獨立任務實例，狀態流程與一次性任務相同，差異如下：

| 時間點 | 行為 |
|---|---|
| 截止日到期 | 若審核中：等審核完成後刪除。若未提交：進入補交期。 |
| 補交期結束未提交 | 紀錄失敗，刪除當期實例，自動建立下一期。 |
| 審核通過 | 點數發放，紀錄成功，刪除當期實例，自動建立下一期。 |
| 家長刪除 | 物理刪除，不計入任務紀錄，視勾選決定是否停止後續週期。 |

> ※ 孩子端可能同時看到上一期「審核中」與新一期「進行中」，屬正常顯示。

### 1-4　補交期設定

| 設定層級 | 說明 |
|---|---|
| 家庭層級 | 預設補交天數，預設值為 2 天 |
| 任務層級 | 建立任務時可自訂，覆蓋家庭預設值 |

### 1-5　刪除行為

| 操作 | 結果 |
|---|---|
| 刪除當期，不勾選停止週期 | 只刪當期，下一期照常自動建立 |
| 刪除當期，勾選停止週期 | 當期刪除，停止自動建立後續期 |
| 刪除審核中的任務 | 可以刪除，家長有最高權限 |
| 被刪除的任務 | 物理刪除，不計入成功或失敗紀錄 |

## 二、審核機制

### 2-1　審核模式總覽

| 模式 | 說明 | 初版 |
|---|---|---|
| 半自動 | AI 初審 + 家長複審窗口（48小時），到期自動通過 | ✓ |
| 人工審核 | 家長手動審核，無自動通過 | ✓ |
| 全 AI 審核 | AI 判斷品質（工整度、正確性），通過即給點 | 第二階段 |

> 預設模式：半自動

### 2-2　半自動審核流程

| 步驟 | 說明 |
|---|---|
| ① 孩子提交照片 | 任務進入「已提交（審核中）」狀態 |
| ② AI 初審 | 判斷「有沒有做」，不判斷品質（初版） |
| ③-A　AI 初審通過 | 等待家長複審，進入 48 小時窗口 |
| ③-B　AI 初審未通過 | 顯示未通過原因清單，提示孩子重拍（最多 5 次） |
| ④ 5 次仍未通過 | 保留 5 張照片，自動轉人工審核 |
| ⑤ 家長複審窗口 | 每日 23:59 彙整通知，從通知時間起算 48 小時 |
| ⑥-A　家長主動通過或否決 | 依家長決定發點或不發 |
| ⑥-B　48 小時到期未操作 | 自動通過，點數發放 |

### 2-3　AI 審核邊界（初版）

| 判斷範圍 | 說明 |
|---|---|
| ✓ 判斷「有沒有做」 | 例：書桌乾淨了嗎、作業本有寫字嗎 |
| ✗ 不判斷「做得好不好」 | 例：數學對不對、字夠不夠工整（預留第二階段） |

### 2-4　AI 未通過提示

採固定清單＋圖示方式，同時列出所有可能原因，讓孩子自行對照（非順序提示）：

- 畫面模糊，請重拍
- 請確認拍到完整內容
- 內容與任務不符，請確認後重拍

### 2-5　家長通知與複審

| 項目 | 說明 |
|---|---|
| 通知時間 | 每日 23:59，彙整當日所有待複審項目 |
| 複審窗口 | 48 小時，從收到通知時開始計算 |
| 顯示方式 | 每筆顯示個別截止時間（同一孩子可能有多筆不同倒數） |
| 家長操作 | 通過 / 否決（兩個方向皆可操作） |

## 三、週期任務時間邏輯

### 3-1　週期種類

| 週期 | 截止日設定 | 說明 |
|---|---|---|
| 每日 | 固定 23:59 | 無需額外設定 |
| 每週 | 家長指定星期幾 | 固定當天 23:59 |
| 每月 | 家長指定幾號 | 見下方月份邏輯 |

> ※ 每學期週期取消，可用一次性任務（挑戰 / 臨時）自行設定截止日替代。

### 3-2　每月截止日邏輯

| 家長設定 | 實際截止日 |
|---|---|
| 1 ~ 29 日 | 每月同日；該月若無此日（如 2 月無 29 日），則抓當月最後一天 |
| 30 或 31 日 | 一律抓每月最後一天（31 日、30 日、28/29 日） |

### 3-3　下一期建立時機

| 情境 | 下一期建立時機 |
|---|---|
| 截止日到期，尚有審核中 | 截止日到期時立即建立下一期，審核中的上一期繼續存活 |
| 截止日到期，未提交 | 進入補交期，補交期結束後建立下一期 |
| 審核通過 | 通過後立即建立下一期 |

## 四、資料模型

> 以下為邏輯決策對應的資料結構，欄位細節待開發階段確認。

### 4-1　Task（任務實例）

| 欄位 | 說明 |
|---|---|
| id | 任務唯一識別碼 |
| family_id | 所屬家庭 |
| created_by | 建立者（家長 user_id） |
| assigned_to[] | 指派對象（孩子 user_id 陣列，支援團隊任務） |
| title | 任務名稱 |
| points | 完成點數 |
| type | once / recurring |
| status | active / submitted / approved / failed / deleted |
| due_date | 截止日 |
| grace_days | 補交天數（覆蓋家庭預設值） |
| grace_due_date | 截止日 + 補交天數 |
| review_mode | manual / semi-auto |
| ai_attempt_count | AI 審核失敗次數（上限 5） |
| review_deadline | 家長複審截止時間（23:59 通知後 +48hr） |
| recurrence_id | 指向 Recurrence；null = 一次性任務 |
| recurrence_seq | 第幾期 |

### 4-2　Recurrence（週期設定）

| 欄位 | 說明 |
|---|---|
| id | 週期設定唯一識別碼 |
| family_id | 所屬家庭 |
| template | 任務範本（title, points, review_mode 等） |
| frequency | daily / weekly / monthly |
| anchor_day | 週：星期幾 / 月：幾號 |
| is_active | 是否繼續產生下一期 |

### 4-3　Submission（提交紀錄）

| 欄位 | 說明 |
|---|---|
| id | 提交唯一識別碼 |
| task_id | 對應任務 |
| submitted_by | 提交者（孩子 user_id） |
| submitted_at | 提交時間 |
| photos[] | 照片陣列（最多 5 張） |
| ai_result | pass / fail / pending |
| ai_feedback[] | AI 未通過原因清單 |
| final_result | approved / rejected |
| reviewed_by | 家長 user_id 或 auto（自動通過） |
| reviewed_at | 審核完成時間 |

### 4-4　Transaction（點數交易紀錄）

| 欄位 | 說明 |
|---|---|
| id | 交易唯一識別碼 |
| family_id | 所屬家庭 |
| user_id | 收點的孩子 |
| amount | 點數 |
| type | task_reward / manual_grant / redemption |
| source_id | task_id；手動給點時為 null |
| note | 手動給點名目；或 Task 刪除時的 title 快照 |
| created_by | 操作者（家長 user_id） |
| created_at | 交易時間 |

> ※ Task 物理刪除時，將 title 快照至對應 Transaction 的 note 欄位，確保點數歷史可追溯。

## 五、決策總覽

| 決策項目 | 結論 |
|---|---|
| 草稿狀態 | 不需要，建立即發佈 |
| 週期任務實例化 | 每期為獨立任務實例 |
| 審核中到期 | 等審核完成再走刪除流程 |
| 補交期設定 | 任務層級自訂；家庭層級設預設值（預設 2 天） |
| 初版任務操作 | 建立、刪除（無暫停） |
| 預設審核模式 | 半自動 |
| 家長複審窗口 | 48 小時，從每日 23:59 彙整通知開始算 |
| AI 初審失敗提示 | 固定清單＋圖示，同時列出所有可能原因 |
| AI 失敗閾值 | 5 次後轉人工，保留 5 張照片供家長判斷 |
| 全 AI 審核 | 列為第二階段，初版不實作 |
| 週期種類 | 每日、每週、每月（取消每學期） |
| 月截止日（1–29） | 每月同日；該月無此日則抓最後一天 |
| 月截止日（30–31） | 每月最後一天 |
| 下一期建立時機 | 截止日到期即建立，不等審核結果 |
| 團隊任務 | 共用 task_id，各自獨立 Submission，全員通過後統一發點 |
| 手動給點 | 獨立 Transaction，不走任務流程 |
| Task 刪除方式 | 物理刪除，title 快照至 Transaction note |
