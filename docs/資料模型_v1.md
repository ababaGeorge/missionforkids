## 一、帳號與家庭結構

### 決策摘要

| 項目 | 決策 |
|------|------|
| 邀請權限 | 任何 parent 角色都可邀請新成員 |
| 家長帳號 | 不帶點數，重新被邀請即可跨家庭 |
| 孩子帳號 | 可跨家庭，歷史紀錄跟著 User |

### Schema

```typescript
import { Timestamp } from 'firebase/firestore';

export interface User {
  id: string;
  displayName: string;
  avatarUrl: string | null;
  authProvider: 'apple' | 'google';
  authProviderId: string;
  roleType: 'parent' | 'child';
  birthday: Timestamp | null;      // 孩子必填，家長 null
  createdAt: Timestamp;
}

export interface Family {
  id: string;
  displayName: string;
  createdBy: string;               // userId
  createdAt: Timestamp;
}

export interface FamilyMembership {
  id: string;
  familyId: string;
  userId: string;
  role: 'parent' | 'child';
  status: 'active' | 'pending' | 'removed';
  invitedBy: string;               // userId（哪個家長邀請的）
  joinedAt: Timestamp | null;      // pending 時為 null
}
```

## 二、點數系統

### 決策摘要

| 項目 | 決策 |
|------|------|
| 點數錢包 | 跟著 Family，每個家庭獨立 |
| 點數過期 | 不過期 |
| 孩子互轉 | 不開放 |
| 跨家庭繼承 | 不自動，由家長用「直接給點」手動補 |
| 扣點時機（兌換） | 孩子下單時即扣 |
| 取消訂單退點 | 可取消，即退點 |

### Schema

```typescript
export interface PointWallet {
  id: string;
  userId: string;                  // 孩子
  familyId: string;
  balance: number;
  updatedAt: Timestamp;
}

export type PointTransactionSourceType =
  | 'task_completion'
  | 'parent_grant'
  | 'reward_order'
  | 'reward_refund';

export interface PointTransaction {
  id: string;
  walletId: string;
  delta: number;                   // 正數（獎勵）或負數（兌換）
  sourceType: PointTransactionSourceType;
  sourceId: string | null;         // taskInstanceId 或 orderId；手動給點時為 null
  createdBy: string | null;        // 家長 userId（parent_grant）或 null
  note: string | null;             // 家長給點名目；或 Task 刪除時的 title 快照
  createdAt: Timestamp;
}
```

## 三、任務系統

### 決策摘要

| 項目 | 決策 |
|------|------|
| 草稿狀態 | 不需要，建立即發佈 |
| 週期種類 | 每日、每週、每月（取消每學期） |
| 指派類型 | individual（特定孩子）或 family（所有孩子各自完成） |
| 預設審核模式 | 半自動（AI 初審 + 家長 48 小時複審窗口） |
| AI 失敗閾值 | 5 次後轉人工，保留所有照片供家長判斷 |
| 補交期 | 任務層級自訂；家庭層級設預設值（預設 2 天） |
| Task 刪除方式 | 物理刪除，title 快照至 Transaction note |

### TaskInstance 展開規則

| 觸發點 | 行為 |
|--------|------|
| Task 建立時 | 為指定孩子建立當期 Instance；family 指派則為所有 active 孩子各建一筆 |
| 新孩子加入家庭時 | 不自動複製任務；家長手動建立時可套用「任務範本」（從過去 Task 紀錄預填） |
| 每期結束時（系統排程） | 自動為符合條件的孩子產生下一期 Instance（Task 仍 active 且孩子仍在家庭中） |

### Schema

```typescript
export type TaskFrequency = 'once' | 'daily' | 'weekly' | 'monthly';
export type TaskReviewMode = 'ai' | 'semi_auto' | 'manual';
export type TaskAssigneeType = 'individual' | 'family';
export type TaskStatus = 'active' | 'archived';

export interface Task {
  id: string;
  familyId: string;
  title: string;
  points: number;
  frequency: TaskFrequency;
  startDate: Timestamp;
  dueDate: Timestamp;
  reviewMode: TaskReviewMode;
  assigneeType: TaskAssigneeType;
  assigneeUserId: string | null;   // individual 時填入，family 時 null
  status: TaskStatus;
  createdBy: string;               // 家長 userId
  createdAt: Timestamp;
}

export type TaskInstanceStatus =
  | 'pending'
  | 'submitted'
  | 'approved'
  | 'rejected'
  | 'missed';

export interface TaskInstance {
  id: string;
  taskId: string;
  userId: string;                  // 永遠綁定特定孩子
  familyId: string;                // 冗餘欄位，方便跨任務查詢
  periodStart: Timestamp;
  periodEnd: Timestamp;
  gracePeriodEnd: Timestamp;       // periodEnd + 補交天數，直接存避免重複計算
  status: TaskInstanceStatus;
  reviewedBy: string | null;       // userId（人工）或 null（AI 或自動）
  reviewedAt: Timestamp | null;
  pointsAwarded: number | null;
}

export type AiResult = 'pass' | 'fail' | 'uncertain';

export interface TaskSubmission {
  id: string;
  taskInstanceId: string;
  familyId: string;                // 冗餘欄位，方便家長審核頁查詢
  submittedBy: string;             // 孩子 userId
  photoUrls: string[];             // 最多 5 張，限制在應用層控制
  aiResult: AiResult | null;
  aiConfidence: number | null;     // 0~1
  submittedAt: Timestamp;
}
```

## 四、商城系統

### 決策摘要

| 項目 | 決策 |
|------|------|
| 商品範圍 | 家庭層（只有該家庭看得到） |
| 商品類型 | 實體（玩具、零食）＋ 虛擬（遊戲時間、選電影） |
| 扣點時機 | 孩子下單時即扣 |
| 取消訂單 | 可取消，即退點（reward_refund） |
| 自動完成窗口 | 家長標記交付後 72 小時，孩子未確認則自動完成 |

### Schema

```typescript
export type RewardItemType = 'physical' | 'virtual';
export type RewardItemStatus = 'active' | 'archived';

export interface RewardItem {
  id: string;
  familyId: string;
  title: string;
  description: string | null;
  pointCost: number;
  itemType: RewardItemType;
  status: RewardItemStatus;
  createdBy: string;               // 家長 userId
  createdAt: Timestamp;
}

export type RewardOrderStatus =
  | 'pending'
  | 'approved'
  | 'delivered'
  | 'completed'
  | 'cancelled'
  | 'rejected';

export interface RewardOrder {
  id: string;
  familyId: string;
  itemId: string;
  userId: string;                  // 孩子
  pointCostSnapshot: number;       // 下單當下點數快照，防止改價影響舊單
  status: RewardOrderStatus;
  cancelledAt: Timestamp | null;
  approvedAt: Timestamp | null;
  deliveredAt: Timestamp | null;
  completedAt: Timestamp | null;   // 孩子確認或逾時自動
  autoCompleteAt: Timestamp | null; // deliveredAt + 72hr，排程用
  createdAt: Timestamp;
}
```

## 五、成就系統

### 決策摘要

| 項目 | 決策 |
|------|------|
| 成就形式 | 統計數據（完成率、連續天數、點數趨勢）＋ 明確徽章 |
| 統計範圍 | 全域，跨家庭累計，跟著 User |

### Schema

```typescript
export type AchievementConditionType =
  | 'streak'
  | 'total_points'
  | 'total_tasks';

export interface UserStats {
  userId: string;                  // document id = userId
  totalTasksCompleted: number;
  currentStreakDays: number;
  longestStreakDays: number;
  totalPointsEarned: number;       // 現有各家庭錢包餘額加總
  updatedAt: Timestamp;
}

export interface Achievement {
  id: string;
  key: string;                     // e.g. 'streak_7' | 'points_100'
  title: string;
  description: string;
  iconUrl: string;
  conditionType: AchievementConditionType;
  conditionValue: number;
}

export interface UserAchievement {
  id: string;
  userId: string;
  achievementId: string;
  unlockedAt: Timestamp;
}
```
